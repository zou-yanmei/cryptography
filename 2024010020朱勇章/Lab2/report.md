# 实验报告：流密码重用攻击分析

## 1. 实验任务
利用流密码重用漏洞，对给定的多组密文进行攻击，恢复出明文信息。

## 2. 攻击原理
### 2.1 数学基础
流密码的加密过程为：$C = P \oplus K$，其中 $C$ 是密文，$P$ 是明文，$K$ 是密钥流，$\oplus$ 表示异或运算。

当使用**相同的密钥流** $K$ 加密两个不同的明文 $P_1$ 和 $P_2$ 时，得到两个密文 $C_1$ 和 $C_2$：
- $C_1 = P_1 \oplus K$
- $C_2 = P_2 \oplus K$

将两个密文进行异或运算：
$$C_1 \oplus C_2 = (P_1 \oplus K) \oplus (P_2 \oplus K) = P_1 \oplus P_2$$

**关键点**：密钥流 $K$ 被抵消了。攻击者虽然不知道密钥，但获得了两个明文的异或结果 $P_1 \oplus P_2$。

### 2.2 英文文本特征
英语文本中，**空格字符（`0x20`）** 的出现频率最高（约占15%-20%）。
- **情况 A**：如果明文 $P_1$ 的某位置是字母（例如小写字母 `a`），明文 $P_2$ 的同一位置是空格（`0x20`）。
  - 异或结果：`a` $\oplus$ `0x20` $=$ `A`（大小写翻转）。
  - 特征：$C_1 \oplus C_2$ 的结果会是一个可读的字母。
- **情况 B**：如果两个明文在该位置都是空格。
  - 异或结果：`0x20` $\oplus$ `0x20` $=$ `0x00`。
  - 特征：$C_1 \oplus C_2$ 的结果会是 `0x00`。

通过扫描 $C_1 \oplus C_2$ 的结果，寻找**可读字母**或**0x00**，即可推断出空格的位置。

### 2.3 攻击步骤
1. **获取密钥流**：一旦确定了 $P_1$ 的某个位置是空格，就可以通过公式 $K = C_1 \oplus 0x20$ 计算出该位置的密钥流。
2. **解密所有密文**：利用已知的密钥流片段 $K$，可以解密所有使用该密钥流加密的密文在该位置的字符：$P = C \oplus K$。
3. **迭代扩展**：利用已知的明文片段（如 "The", "is", "of"），反推出更多的密钥流，进而解出更多的明文，像拼图一样逐步还原完整内容。

## 3. 实验过程
1. **数据准备**：从题目中提取所有密文的十六进制字符串。
2. **代码实现**：编写 Python 脚本（`attack.py`）。
   - 将十六进制密文转换为字节流。
   - 两两计算密文的异或值。
   - 遍历异或结果，如果发现可打印字符或符合空格特征的值，则假设该位置存在空格。
   - 根据空格假设推导出密钥流，并尝试解密所有密文。
3. **结果分析**：观察解密输出。由于脚本是自动化的，输出可能包含部分乱码（`?` 或 `.`）。在实际操作中，需要根据已解出的单词片段（如 "Secr" 推断 "Secret"），手动修正并补全密钥，直到完全还原明文。

## 4. 实验结论
本次实验验证了**密钥重用**在流密码中的严重危害。即使攻击者不知道密钥，只要获得两段使用相同密钥加密的密文，利用自然语言的统计特征（空格），就能有效地恢复出明文。这说明流密码必须严格遵循“一次一密”的原则，或者使用安全的密钥派生机制，绝不能重复使用密钥流。
