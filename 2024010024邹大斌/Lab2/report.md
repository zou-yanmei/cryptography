## 一、攻击核心逻辑
本次实验采用流密码密钥重用唯密文攻击，核心逻辑：
1. **密码学原理**：所有密文使用同一密钥流 \(K\) 加密，满足 \(C_i = M_i \oplus K\)；
2. **密钥消除**：密文两两异或可直接消去密钥流，得到 \(C_i \oplus C_j = M_i \oplus M_j\)；
3. **攻击核心**：利用**英文明文均为可打印 ASCII 字符（32~126）**的特性，暴力枚举每个位置的密钥字节，验证所有密文对应明文是否为可打印字符，唯一满足条件的即为正确密钥字节；
4. **辅助特性**：空格(0x20)与字母异或会互换大小写，进一步验证明文合理性。

## 二、密钥流推断过程
1. **数据转换**：将11段十六进制密文转换为字节数组，方便逐字节运算；
2. **逐位置枚举**：遍历目标密文的每个字节位置，枚举0~255的候选密钥；
3. **可打印验证**：对每个候选密钥，计算所有密文在该位置的明文，校验是否全为可打印字符；
4. **构建密钥流**：保存每个位置的正确密钥字节，生成完整密钥流；
5. **解密**：用密钥流与目标密文异或，得到明文。

### 2. 核心结论
流密码的核心安全准则：**同一密钥绝对不能使用多次**，否则攻击者可通过密文异或轻松消除密钥流，还原明文。
