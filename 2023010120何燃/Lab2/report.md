# Lab 2: 流密码多次填充攻击实验报告

## 1. 采用的分析方法
本次实验使用了基于**空格启发式（Space Heuristic）**的已知明文攻击策略：
由于流密码在相同密钥下满足 $C_1 \oplus C_2 = M_1 \oplus M_2$，我们利用了 ASCII 编码中空格字符（`0x20`）的特性：**当空格与任意英文字母（大小写均可）进行 XOR 运算时，其结果会翻转第 6 个比特位，变成与其大小写相反的合法英文字母**（结果落在 `[0x41-0x5A]` 或 `[0x61-0x7A]` 范围内）。

通过逐列扫描所有密文的字节，若某密文在该列与绝大多数其他密文异或的结果均为合法英文字母，即可断定该密文在该位置的明文是空格（`0x20`）。据此，可通过公式 $K = C \oplus \text{0x20}$ 逆向推导出该位置的单字节密钥流。

## 2. 目标密文内容的确认方式
通过上述自动化空格分析法，能够成功推断出大约 90% 的密钥流。将这部分密钥流应用到目标密文上，可以得到一段带有缺失字符（乱码）的残缺明文，如 `The secret m*ssage is...`。

由于明文是自然英语句子，我采用了**词法推断与上下文猜测（Crib Dragging）**的方法。根据残缺单词和前后文语境（如猜测 `m*ssage` 为 `message`），补全了剩余缺失的明文字母。最后，通过 $K = C_{target} \oplus M_{guessed}$ 反推出剩余部分的密钥，并验证该密钥解密其他 10 段密文是否符合英语语法，从而彻底确认了目标密文的内容。

## 3. 最终解密得到的明文
目标密文的完整解密结果为：
**The secret message is: When using a stream cipher, never use the key more than once**
