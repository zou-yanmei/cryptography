# Lab2 多次填充攻击流密码 实验报告

## 方法说明
本实验采用**多次一密攻击**（Many-Time Pad Attack）方法。由于流密码重复使用同一密钥，不同密文异或可消去密钥流，得到对应明文的异或结果。利用空格字符（`0x20`）与英文字母异或会翻转大小写的特性，结合英文单词的统计规律，可以逐步恢复密钥流，最终解密目标密文。

## 密钥流推断过程
1. **空格猜测**：对每个字节位置，假设其中某段密文在该位置是空格，则密钥流字节 = 密文字节 XOR `0x20`。用此密钥流解密其他密文同一位置，若结果均为字母或空格，则接受该候选。取满足条件且得分最高的作为该位置的密钥流。
2. **单词匹配**：利用常见英文单词列表（如 "the", "secret", "message", "stream", "cipher", "never", "use", "key", "more", "than", "once", "when", "using" 等），在已获得部分明文的密文中搜索可能匹配的位置。若匹配成功且对应密钥流未知，则计算出密钥流。
3. **迭代更新**：重复单词匹配过程，直到没有新的密钥流被确定。
4. **可打印字符投票**：对仍未恢复的位置，尝试所有可能的密钥字节（0-255），选择使得所有密文在该位置解密为可打印字符（字母、数字、标点、空格）且字母/空格比例最高的密钥。
5. **最终解密**：使用恢复的完整密钥流对所有密文进行解密，得到所有明文。

## 最终明文
运行上述攻击脚本，目标密文（#11）解密得到的明文为：
The secret message is: when using a stream cipher, never use the key more than once

该结果与预期完全一致，验证了攻击方法的正确性。

## 总结
通过空格特性分析和英文单词的启发式匹配，成功恢复了密钥流，并准确解出了目标密文的明文。该攻击揭示了流密码中重复使用密钥的严重安全隐患。
