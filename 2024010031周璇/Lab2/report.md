# Lab2 多次填充攻击流密码实验报告
## 1. 使用的分析方法
我使用的是**多次填充攻击（Two-time Pad Attack）**，核心利用流密码的安全漏洞：如果同一密钥流被重复使用加密多段明文，攻击者无需知道密钥即可完成破解。
具体实现步骤：
1.  将所有使用同一密钥流加密的密文两两异或，消去密钥流，得到「明文1 XOR 明文2」的结果。
2.  利用实验提示的规律：空格字符（0x20）和英文字母异或时，会翻转字母的大小写，结果仍落在英文字母的ASCII范围内。通过这个规律统计每个字节位置是空格的概率。
3.  对高概率的空格位置，假设该位置的明文是空格，反推出该位置的密钥流字节（密钥流 = 密文字节 XOR 0x20）。
4.  验证反推的密钥流：用该字节解密其他密文的对应位置，结果必须是可打印的英文字符，确保密钥流的合理性。
5.  用最终还原的密钥流，解密目标密文，得到明文。

## 2. 确认目标密文明文的过程
1.  首先通过统计每个位置的空格得分，筛选出高概率的空格位置，反推得到初步的密钥流。
2.  用初步的密钥流解密目标密文，得到可识别的英文片段，比如开头的"The secret message is:"，符合英文句子的语义。
3.  结合英文的语法和单词拼写，补全剩余的密钥流字节，确保解密后的内容是完整、通顺的英文句子，无乱码。
4.  最终验证：用还原的完整密钥流解密所有密文，得到的都是通顺的英文句子，说明密钥流完全正确，目标密文的解密结果也正确。

## 3. 解密得到的明文
The secret message is: When using a stream cipher, never use the key more than once
