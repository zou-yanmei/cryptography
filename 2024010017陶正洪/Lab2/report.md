# 多次填充攻击流密码实验报告

## 1. 分析方法
本次实验采用**多次填充攻击（Many-Time Pad Attack）**，针对流密码重复使用同一密钥的安全漏洞进行破解，核心步骤如下:
1.  **消去密钥流**：流密码加密公式为 `C_i = M_i ⊕ PRG(k)`，若两段密文使用同一密钥加密，则 `C_i ⊕ C_j = M_i ⊕ M_j`，密钥流被完全消除，无需知道密钥即可分析明文。
2.  **空格位置推断**：利用ASCII码特性——空格（`0x20`）与字母异或时，会翻转字母的大小写（大写↔小写），结果仍为字母。通过统计异或结果中字母的占比，推断明文中的空格位置。
3.  **迭代补全密钥流**：通过空格位置计算出部分密钥流字节后，用已知密钥流解密其他密文，验证并推断更多密钥流字节，最终补全完整密钥流。
4.  **解密目标密文**：用完整的密钥流与目标密文异或，得到最终明文。

## 2. 密钥流推断过程
1.  **初始推断**：将目标密文与其他10段密文逐位异或，统计每个位置异或结果为字母的次数，超过半数则判定目标密文该位置为空格，通过 `k = C ⊕ 0x20` 计算出对应密钥流字节。
2.  **迭代补全**：遍历所有未知位置，假设某段密文的该位置为空格，计算候选密钥流，验证其是否能让所有密文的该位置解密为可读ASCII字符，选择票数最高的候选作为该位置的密钥流。
3.  **终止条件**：当一轮迭代中没有新的密钥流字节被确定时，停止迭代，得到完整密钥流。

## 3. 最终解密结果
解密得到的目标密文明文为：
The secret message is: A stream cipher key should never be reused, as it breaks the security completely.

## 4. 结果验证
- 解密后的明文为完整可读的英文句子，符合自然语言特征，无乱码或不可读字符。
- 密钥流与所有密文异或后，得到的明文均为可读的英文文本，验证了密钥流的正确性。
- 明文内容符合流密码的安全原则，与实验背景的安全前提呼应，验证了解密结果的合理性。
